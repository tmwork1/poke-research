# ページング方式の使い分け

このサイトでは、画面ごとに「無限スクロール」と「ページネーション」を意図的に使い分けている。

- **トップページ（`/`）: 無限スクロール**
  新着記事を上から流し読みし、新しい情報をチェックする回遊用途が中心のため、スクロールだけで読み進められる無限スクロールを採用する。追加読み込みのたびに `history.replaceState` で `?page=N` をURLへ反映し、記事リンクを開いた後に「戻る」で戻ってきたときにおおよその位置を保てるようにしている。ただし `?page=N` を直接開いた場合はそのページ単体を描画するだけ（1〜N ページ分の再現はしない）なので、その場合は「1ページ目から表示」リンクを添えて軽量に案内する。

- **`/items`・タグ絞り込みページ: ページネーション**
  検索・絞り込み結果を誰かと共有したり、後で同じ位置に戻ってきたりする用途を想定し、URLの `page` パラメータがそのままページ全体の状態を表す通常のページネーションを使う。クローラーが個々のページを独立したURLとして巡回できる点もSEO上の利点になる。

この使い分けにより、「回遊・新着チェックはトップの無限スクロールで」「検索結果の位置共有やSEOは `/items` のページネーションで」という役割分担を保つ。
