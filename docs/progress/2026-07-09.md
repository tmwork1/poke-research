# 進捗メモ 2026-07-09

## 新着記事のX投稿下書きをDiscordに通知（`feat/new-items-x-draft-digest`）

cronで新着記事（AIレビューで採用され新規insertされた記事）が見つかったとき、Xに投稿しやすい形式の下書き文をまとめてDiscordへ送る機能を追加。

- 対象は収集系cronジョブ全て（Qiita/Zenn/note/Blog(Brave Search)/RSSフィード追従/はてなブックマーク）。link-check・週次DBレビューは対象外（新着記事を扱わないため）。
- 下書きの元データはAIレビュー（`article-ai.ts`）が既に生成しているsummary/tagsを流用し、新規のOpenAI呼び出しは追加していない。
- 送信先は既存の `ALERT_WEBHOOK_URL`（運用アラート・週次レポートと同じDiscordチャンネル）をそのまま使用。新しい環境変数は追加していない。
- 複数件見つかった場合は1件ずつのメッセージではなく、まとめて1つのDiscordメッセージに列挙する。

### 実装
- `src/lib/importers/process-import-item.ts`: `ImportReviewOutcome`/`ImportItemOutcome` に任意項目 `summary`/`tags` を追加し、採用時の outcome に含めるようにした（既存の呼び出し元・型は破壊的変更なし、追加項目のみ）。
- `src/lib/notify.ts`: `sendNewItemsDigest()` を追加。`action==='inserted'` の記事から `タイトル＋要約冒頭80文字＋URL＋タグ由来のハッシュタグ最大3つ` の下書きを組み立て、Discordへ1メッセージで送る。
- `src/worker.ts`: 各cronハンドラ（Qiita/Zenn/note/Blog/Feed/Hatena）の成功ログ直後に、`action==='inserted'` の記事があれば `sendNewItemsDigest` を呼ぶ `notifyNewItems()` ヘルパーを追加。

### 検証
- `npm test`（96件）・`npm run build` green。
- `tests/process-import-item.test.ts` の既存テストが `summary`/`tags` フィールド追加で `deepEqual` 不一致になったため、期待値を更新。
