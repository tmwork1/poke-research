# 進捗メモ 2026-07-04

## 今日の確認結果
- ロードマップは初期構想の主要要素と概ね整合している。
- 画面作成は、以前は「簡易 UI」までの記載だったが、トップ・一覧・詳細・検索結果の作成を明示した。
- 全体進捗が分かるように、ロードマップへマイルストーン別の状況表を追加した。
- さらに、各マイルストーンの下に実作業ベースの詳細計画を追加し、次に着手する粒度まで落とし込んだ。
- M2 はこれから開始する前提に切り替え、ロードマップ上の状態を「進行中」に更新した。
- M2 の画面・検索・API 導線を実装し、`npm run build` が通るところまで確認した。
- これに合わせて、ロードマップ上の M2 状況を「完了」に戻した。
- ロードマップの進捗表示を、`[x]` / `[-]` / `[ ]` のチェックリスト形式に変えて、完了・進行中・未着手が一目で分かるようにした。
- 詳細計画もチェックリスト化して、マイルストーン単位だけでなく作業単位でも進捗を追えるようにした。
- M3 は CSV 取り込みを前提にせず、API 取り込みと収集ジョブに絞る方針へ整理した。
- Qiita 収集を最初の対象として、API インポーターとローカル実行用の収集スクリプトを追加した。
- 収集データの provenance は、Qiita 由来の source/item metadata に残す方針で実装した。
- 最新スキーマに合わせて、廃止済みの `notes` API 参照をロードマップから除いた。
- ローカル接続設定の既定値を、実際の Supabase/Postgres ポートに合わせて `54322` にそろえた。
- `scripts/test-db.mjs` を `sources` / `items` / `annotations` の CRUD スモークテストに更新した。
- ローカル Supabase に開発用権限を付与し、M1 の CRUD スモークテストが成功した。
- これで M1 は完了扱いにできる状態まで到達した。
- AI エージェントの利用モデルを `.env` の `OPENAI_MODEL` で指定する形に整理し、初期値は `gpt5-nano` にそろえた。
- Qiita 取り込みでは、DB 登録前に OpenAI で妥当性チェック、要約、タグ付けを行う流れを入れた。
- AI レビュー処理は Qiita モジュールから切り出し、記事インポーター共通で使える独立モジュールに分離した。
- 検索条件の最適化は後回しにして、まずは収集フローと AI 判定の安定化を優先する方針にした。
- その方針をロードマップにも追記し、M3 は収集フローの安定化を先に進めるよう整理した。
- 収集試行で、`temperature: 0` が一部モデルで受け付けられないことが分かり、OpenAI リクエストから外したうえで Qiita 1 件の収集フローを実際に通した。
- その後、Qiita 側のタグ同期で共有関数の export 漏れが見つかり、修正後は 1 件の記事を `updated` として取り込めた。
- コード内コメントの日本語説明を強化し、catalog / Qiita importer / OpenAI review / API 共通処理の意図が追いやすくなった。
- さらに、主要な ts ファイルの先頭に役割説明を追加し、ファイル単位でも用途が見えるようにした。
- Claude Code 向けの基礎指示として、作業前確認、最小変更、検証、進捗更新のルールを [AGENTS.md](../../AGENTS.md) と [CLAUDE.md](../../CLAUDE.md) にまとめた。
