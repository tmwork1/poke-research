## CIが2026-07-16以降ジョブ0件で即失敗していた問題の修正（PR #81）

Brave Search cron削除のPR（#80）のCIを確認したところ、build/migrationsのどちらのジョブも実行されずワークフロー自体が即失敗していた。調査したところ、2026-07-16のRLS対応（`4255f5c`, PR #79マージ）で`.github/workflows/ci.yml`の`Stub Supabase auth schema`ステップをheredoc化した際、終端行`SQL`が列0（無インデント）になっており、YAMLのブロックスカラー（`run: |`）が要求する最低インデントを満たせず構文エラーになっていた。この結果`main`も含め、2026-07-16 14:43 UTC以降のCI実行がすべてジョブ0件・即失敗になっていた（`gh api .../check-suites`で`conclusion: failure`かつ`latest_check_runs_count: 0`を確認）。3週間近くCIが機能していなかったが、Discord等への通知経路ではないため気づかれていなかった。

heredocをやめ、`psql -c "..."`の複数指定に置き換えて修正（PR #81、CI green確認後マージ）。

## Brave Search API解約に伴うブログ収集cronの削除（実装は維持）

Brave Search APIを解約したため、ブログインポーター（`src/lib/importers/blog.ts`）の自動実行cronを停止した。noteの自動実行停止（403 Access denied、2026-07-09）と同じ扱いで、cronからの呼び出しだけを外し、インポーター本体・手動起動用API（`POST /api/import/blog`）は変更していない。APIキーを再取得すれば、cronの再登録・手動起動のいずれもすぐ行える。

- `wrangler.jsonc`: `triggers.crons`からブログ用エントリ（`"0 1,5,9,13,17,21 * * *"`）を削除。アカウント全体のCron Trigger登録数（上限5件）の空きが2件→3件に増えた。
- `src/worker.ts`: `BLOG_CRON`/`BLOG_ROTATION_SLOT_MS`定数、`scheduled()`内の分岐、`runScheduledBlogImport`関数、および使われなくなったimport（`resolveBlogSyncOptions`/`syncBlogCollection`/`resolveBlogKeywordIndex`/`BLOG_KEYWORDS`）を削除。
- `docs/scheduled-jobs.md`: 登録済みCron Trigger表からブログの行を削除し、廃止の経緯を追記。
