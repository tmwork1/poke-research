## 環境変数の運用変更（`.env` → ローカル、`.env.production` → 本番）

ユーザーの判断で、ローカル環境変数の置き場所が変更された。

- `.env`: ローカル（`127.0.0.1:54321` の Supabase、`127.0.0.1:54322` の Postgres）。`astro dev` と Node スクリプト（`node --env-file=.env ...`）の両方がこちらを見る。
- `.env.production`（新規、`.gitignore` 済み）: 本番 Supabase の接続情報。本番向けの一回限りの操作（`npm run release`、`scripts/db/*.mjs` のメンテナンススクリプト、今回のバックフィルなど）はこちらを明示的に使う。
- `.dev.vars`: 従来通りローカル値のまま。Cloudflare Vite プラグイン（`astro dev` の Miniflare エミュレーション）が読む。

README.md・`docs/reference/operations.md`・`scripts/db`/`scripts/eval` 配下9ファイルのヘッダーコメントには、まだ「本番操作は `node --env-file=.env`」という旧前提の記述が残っている。ユーザーの指示で今回は把握のみに留め、編集は未実施（次回以降の対応候補）。

## QIITA_TOKEN を本番シークレットに反映

ユーザーが取得した Qiita トークンは `.env`/`.dev.vars`/`.env.production` に追加済みだったが、本番 Cloudflare Workers には未登録だった（`npx wrangler secret list` で確認）。ローカルの `/api/import/qiita` は `cloudflare:workers` の `env`（`.dev.vars` 経由）で既に有効に使えていたが、本番の日次 Qiita cron は無トークン（未認証、60回/時制限）のまま動いていた。

- `npx wrangler versions secret put QIITA_TOKEN` でシークレットを追加（`wrangler secret put` は現行バージョンが未デプロイ状態のため使用不可だった）。
- `npx wrangler versions deploy <version-id>@100 --yes` で100%トラフィックにデプロイし反映。

## Qiita/Zenn の2020年遡及バックフィル（本番実行）

[2026-07-07](2026-07-07.md) でローカル `astro dev`（Miniflare）が長時間フェッチで不安定だったため保留していた一度きりのバックフィルを、本番（`https://poke-research.com`）に対して実行した。

- 対象: Qiita（`created:` 範囲で年別クエリ、2020〜2026年、計7クエリ）、Zenn（`pages=3`、topic=pokemon 全件）。note・はてなブックマークは今回除外（日付/件数を確実に絞り込めないため）。
- 資格情報は `.env.production` の `ADMIN_USERNAME`/`ADMIN_PASSWORD`（Basic認証）・`QIITA_TOKEN` を使用。undici の既定5分ヘッダタイムアウトは `Agent({ headersTimeout: 0, bodyTimeout: 0 })` で無効化（`scripts/collect/backfill.mjs` と同じパターン）。
- 一時スクリプト（`.tmp-2020-backfill-prod.mjs`、プロジェクトルート直下、実行後削除）で年別に順次POSTし、結果をログした。
- **結果**: ローカルでは繰り返し失敗していた長時間リクエストが、本番では全件エラー無く完了（Qiita 7クエリ・Zenn 1クエリ、計8リクエスト）。
  - Qiita: 2020=77件（新規5）、2021=41件（新規5）、2022=76件（新規4）、2023=58件（新規5）、2024=67件（更新5）、2025=179件（更新5）、2026=62件（更新3）。
  - Zenn: 69件（更新5、新規0＝既存270件と全件重複）。
  - 合計: fetched=629, inserted=19, updated=18, skipped=592（棄却理由の大半はAIレビューでの主題不一致、または既存記事との重複）。
- 実行後の本番 items 件数: 305件（実行前270件 + 新規19件 + 別途累積分）。

### 教訓
- ローカル `astro dev`（Miniflare/`@cloudflare/vite-plugin`）は、数百件規模・数分単位のAIレビューを伴う長時間単発リクエストで `fetch failed` により不安定になることを確認済み（[2026-07-07](2026-07-07.md)）。同種の大規模一回限りバッチは、最初から本番相手に実行する前提で計画した方がよい。
